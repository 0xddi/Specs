# Техническое задание

## Основная задача и общие сведения
Создание лёгкого фронтенд WEB-приложения под названием GopherNote для хранения и управления заметками в Markdown разметке. Ориентирами являются известные приложения по типу Samsung Notes и Evernote, уникальными функциями которого являются система голосового ввода и система отображения мотивационных цитат в зависимости от контекста.

## Конкретный функционал приложения
- Возможность выбрать тип заметки: "Текст" или "Список"  
- CRUD-функционал
- Возможность закрепления заметки для быстрого доступа к ней  
- Возможность назначать теги и выполнять поиск заметок по ним  
- Возможность выбрать цвет для заметки

- Возможность голосового ввода содержимого заметки (при этом с одновременной возможностью ручной печати)

## Модули обработки и визуализации данных

### Модуль MD-разметки (Markdown)
- **Назначение:** Трансформация сырого текстового контента заметки в валидный, безопасный HTML-код для его последующего отображения в режиме реального времени.
- **Спецификация синтаксиса:** Полная поддержка заголовков (`#`–`######`), стилизации текста (`**жирный**`, `*курсив*`, `~~зачеркнутый~~`), маркированных и нумерованных списков, блоков кода, цитат (`>`) и горизонтальных линий (`---`).

### Модуль голосового ввода (Speech-to-Text)
- **Технологическая основа:** Браузерный Web Speech API, вызываемый из среды C# через механизм JavaScript Interop.

### Модуль контекстных цитат (QuoteService)
- **Алгоритм анализа:** Мониторинг состояния локального хранилища и логов активности пользователя (также можно реализовать через технику из геймдева, где за каждое действие начисляются "очки" в отдельных переменных).
- **Поведенческие триггеры:**
  - Высокая активность (массовое создание заметок за короткий промежуток времени) -> Выдача цитат о предотвращении переутомления и важности отдыха.
  - Низкая активность (отсутствие новых записей, редкие сессии) -> Выдача триггерных цитат, мотивирующих преодолеть лень и прокрастинацию.

## Архитектура хранения данных (Локальная БД)
- **Общая концепция:** Использование системного объекта `localStorage` браузера пользователя. Приложение работает полностью автономно на стороне клиента без отправки контента на внешние сервера.
- **Сериализация:** Переход состояния объекта из памяти WebAssembly в постоянную память устройства осуществляется по цепочке:
  `C# Object (List<Note>)` -> `JsonSerializer.Serialize()` -> `JSON-строка` -> `js.InvokeVoidAsync("localStorage.setItem", ...)`

## Требования к интерфейсу (UI/UX) и вёрстке в целом
- **Стилистика:** Строгий минималистичный монохромный дизайн в темных и светлых тонах ("Notion / Evernote").
- **Splash Screen (Экран загрузки):** Центрированный индикатор загрузки.
- **Карточка заметки (NoteCard):** Ограничение превью текста лимитом в n-ое количество символов с последующим добавлением многоточия. Блок прикрепленных тегов позиционируется строго над кнопкой «Редактировать» и прижимается к нижней части карточки для сохранения ровной сетки при любой длине текста.
